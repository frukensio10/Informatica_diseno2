  • Traducción de nombre de dominio a dirección IP(DNS)

    Esto nos permite traducir un nombre de dominio (www.ciw.cl) a una dirección IP (146.83.4.11) este es el único numero que se necesita para llegar al computador de destino, esto es crucial para que funcione la web, estos nombres deben ser traducido si o si antes de conectarlo al servidor, también se requiere de los servidor de nombres (DNS) que responde por cada dominio y con rapidez.



  • Conexión y Transporte(SOCKET)

    Cuando se obtiene la dirección IP, luego para que podamos enviar y recibir datos esto tiene que pasar por medio del SOCKET que es un protocolo de correcciones de errores, este solo aparece en los extremos de la conexión como el navegador y el servidor, la otra parte de la red no interviene; todo esto es lo contrario al servicio telefónica que es mucha más caro y no económico como el de las computadora.

  • Protocolo HTTP

    Esto sucede cuando el navegador envía una URL al servidor con un contenido almacenado, para que el navegador interprete y decida qué hacer con esto, hay termina el dialogo HTTP; también se establece un código para el usuario solicitante y el que contiene la información solicitada, así esto pueden hablar un mismo idioma a la hora de transmitir información por la red.



  • Ruteos de Paquetes (IP)

    El servicio basico que me debe proveer un ISP es el ruteo de los datos que fluyen entre el navegador y el servidor, estas redes tienen que pasar a traves de varis redes de paises y contienente diferentes, esto es un servicio fundamental del ISP.
